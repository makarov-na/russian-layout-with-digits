## Context

Текущий спек `openspec/specs/winkeys-pd/spec.md` содержит 19 требований (REQ-01 — REQ-19) в одном файле. Среди них:
- 1 требование на регистрацию раскладки
- 10 требований на клавиши цифрового ряда (AE01–AE10)
- 8 требований на Shift+F-клавиши

Все требования уже реализованы и работают. Изменение носит чисто организационный характер.

## Goals / Non-Goals

**Goals:**
- Каждое требование — отдельный спек-файл для независимого отслеживания
- Единообразный формат имён: `req-XX-<short-name>`
- Сохранение содержания требований без изменений

**Non-Goals:**
- Изменение поведения раскладки
- Добавление новых требований
- Изменение реализации (патч, evdev.extras.xml)

## Decisions

**Именование спеков**: `req-XX-<short-name>` — где XX — номер требования (01–19), short-name — краткое описание на английском. Префикс REQ обеспечивает сортировку и однозначную идентификацию.

**Формат файлов**: Каждый спек — `specs/<name>/spec.md` с заголовком `## ADDED Requirements` и одним требованием. Это соответствует формату OpenSpec для новых capabilities.

**Удаление монолитного спека**: После создания всех19 спеков исходный `winkeys-pd` удаляется, так как его содержимое будет полностью перенесено.

- [Размер change] → 19 спек-файлов в одном change. Допустимо для реорганизации.
